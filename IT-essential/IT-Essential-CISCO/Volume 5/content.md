### Физические компоненты сетей
* Узловое/конечное устройство (хост) – принимает или передаёт интерпретированную информацию; 
браузер, IP-телефон etc.
* Промежуточное устройство – передача данных от одного устройства до другого.
* Среда передачи – внутри среды создаётся канал, по которому сообщение доставляется до получателя.


### Промежуточные устройства
1. Коммутатор (**switch**) – подключает несколько устройств к сети; передаёт трафик внутри неё; 
для передачи информации использует физический адрес (mac) и таблицу маршрутизации, состояющую 
из номера порта и MAC адреса.
2. Маршрутизатор (**router**) – пересылает трафик между сетями (глобальными), использует IP 
для передачи информации; есть таблица маршрутизации.
3. Беспроводной маршрутизатор – подключает устройства по Wi-Fi; может содержать коммутатор 
для подключения проводных устройств.
4. Точка доступа – Wi-Fi антенны.
5. Модем – подключает локальную сеть к сети интернет через мобильную связь.


### Топология сетей
#### PAN
Personal Area Network.
Персональная сеть для подключения. Мыши, клавиатуры, наушники etc.
* Небольшой радиус действия;
* Нужна для работы конкретного пользователя с конкретным устройством (bluetooth).


#### LAN
Local Area Network.
* Локальные устройства, соединённые кабелями в пределах небольшой территории;
* Количество компьютеров роли не играет (10, 10'000);
* Управляется одним IT оделом или администратором.


#### VLAN
Virtual Local Area Network.
Сеть 1 для одного отдела, сеть 2 – для другого.
* Группировка пользователей с учётом административной принадлежности;
* Логический способ разделения сетей;
* Позволяет создавать запреты на конкретную сеть, контролировать трафик.


#### WLAN
Wireless Local Area Network.
Беспроводной LAN.
* Беспроводная сеть, радиоволны;
* Небольшая географическая область;
* Управляется одним IT оделом или администратором.


#### WMN
Wireless Mesh Network.
WLAN с повторителями Wi-Fi сигнала.
* Несколько точек доступа.


#### MAN
Municipal Area Network.
* Муниципальная сеть, большая. В пределах города либо квартала.


#### WAN
Wide Area Network.
* Глобальная сеть, объединение локальных сетей из разных локаций;
* Доступ через провайдеров;
* Интернет – самый популярный вариант;
* Есть другие сети – сети военных.


#### VPN
Virtual Private Network.
* Защищённое подключение к частной сети по открытой.


## Технологии подключения к сети Интернет
Когда интернет только появлялся, то скорости были небольшие. Использовались модемы, 
подключавшиеся к телефонным линиям. Сигнал передавался аналоговый.
<br>
Телефон занят, когда подключены к интернету.


### Аналоговый телефон
Данные идут по стандартной телефонной линии, используется аналоговый модем, 
созванивающийся с другим модемом; коммутируемый доступ.


### Цифровая сеть с интегрированными услугами
ISDN – несколько каналов для передачи голоса, видео и данных по телефонным линиям.


### Широкополосный доступ
Одна среда передачи, но разные частоты сигналов;
DSL, ISDN, стпутниковые и мобильные подключения.


## Подключение к интернету
### DSL
Непрерывно доступная служба, то есть нет необходимости каждый раз 
набирать номер для подключения к интернету.


### Кабельная линия
Не нужна телефонная линия, а нужен специальный UDP кабель для подключения.


### Оптоволокно
Используются в крупных корпаративных, магистральных сетях и ЦОД.
Постепенно становится доступно в домашних сетях.


### Беспроводной доступ
Сигнал поступает к пользователю через вышки связи в зоне прямой видимости.
* 900 МГц – 65км;
* 5.7 ГГц – 3км.


### Широкополосная спутниковая линия
Для связи используется спутниковая антенна.
Скорость получения данных = 10/1 Мбит/с


### Сотовая связь
Вышки сотовой связи обеспечивают выход в интернет.


### Раздача Wi-Fi с мобильного устройства
Подключение к интернету происходит за счёт раздачи Wi-Fi.


## Модель TCP/IP
Семейство протоколов TCP/IP используется для сетевого обмена 
данными, адресации, маршрутизации и поддержания надёжности.


### Уровни
2. Физический – LAN, WAN;
2. Сетевой – IP;
3. Транспортный – TCP(Transport Control Protocol), UDP(User Datagram Protocol);
4. Прикладной – TCP: FTP, HTTP(S), SMTP, POP, SSH; UDP: SNMP, DHCP, RTP, DNS.
Некоторые протоколы могут использовать для выполнение разных задач как TCP, так и UDP.


### Протоколы транспортного уровня
* TCP;
* UPD.


#### TCP
Например, SMTP, HTTP.
* Данные разбиваются на пакеты;
* Проверка, что данные дошли. Повторная отправка в противном случае;
* Доставка данных в заданной последовательности;
* Более надёжный, более медленный протокол.


##### Три базовые операции
* Нумерация и отслеживание пакетов;
* Потдверждение получения данных;
* Повторная отправка потерянных данных.


#### UDP
* Допускается потеря данных;
* Нет проверки, что пакеты пришли;
* Низкая надёжность, высокая скорость.

### Модель OSI
TCP/IP и OSI – одно и то же. Это логическое устройство сетей.
Описано, как могу взаимодействовать устройства в сети.


#### Состоит из семи уровней
1. Физический уровень – нули и единицы; отвечает за кодировку, стандарты кабелей;
2. Канальный уровень (**коммутатор**) – адресация по MAC;
3. Сетевой уровень (**маршрутизатор**) – адресация по IP;
4. Транспортный уровень – переход к протоколам; TCP и UPD; адресация по «номеру порта»;
5. Сеансовый уровень – установка соединения;
6. Уровень представления – работа с форматами файлом, их спецификация; подготовка даннных для следующего уровня;
7. Уровень приложений – браузер, торрент etc.

## Клиент-сервер
### Клиент
* Использование сервиса;
* Клиентский софт;
* Запросы к серверу.


### Сервер
* Предоставление сервиса;
* Серверный софт;
* Обработка запросов клиентов;
* Несколько клиентов.

### DNS
* Клиент: Домен в адресной строке -> DNS;
* DNS: IP сервера -> клиент;
* Клиент: запрос на сервер по IP.


### Файловый сервер
FTP, но он небезопасен, поэтому:
* FTPS (file transfer protocol secure);
* SFTP (SSH file transfer protocol) – основан на SSH;
* SCP (secure copy) – основан на SSH.


### WEB-сервер
* Клиент: HTTP запрос -> сервер;
* Сервер: обработка запроса, HTML -> клиент;
* Клиент: обработка HTML кода браузером.


#### Почтовый сервер
* SMTP (simple mail transfer protocol);
* POP (post office protocol);
* IMAP (internet message access protocol);
Адресант использует SMTP, адресат обращается к серверу по POP или IMAP.


#### Proxy
* Для хранения кэша веб-страниц, к которым часто обращаются;
* Скрывать IP, т.к. запросы имеют IP самого proxy.


#### Syslog
Сервер, хранящий и обрабатывающий логи. <br>
Протокол syslog поддерживают:
* роутеры;
* свичи;
* etc.

---

Основные функции:
* Сбор информации из системных журналов;
* Выбор типа хранимой информациии;
* Место назначения захваченных syslog сообщений.


### AAA
Authentication, Authorization, Accounting.
* Аутентификация – сопоставление клиента и учётной записи в системе по логину, паролю etc;
* Авторизация – сопоставление персоны (после аутентификации) и полномочий, которые у неё есть;
* Учёт – слежка за потреблением реурсов, логгирование подключений и действий.


## Сетевые устройства
### NIC
NIC (network interface controller) – сетевуха, наиболе часто стандарта Gigabit;
* Проводная;
* Беспроводная;
* По USB.


### Повторители, мосты, концентраторы
#### Повторители
* Используются для регенерации сигнала;
* Промежуточные устройства также являются повторителями;
* Чаще используются для оптоволокна.


#### Концентратор
* Отправляет полученный сигнал на все подключённые устройства;
* Также регенерирует сигнал;
* Устаревшее у-во, заменён свичём.


#### Мосты
* Сегментируют LAN, регистрируют все устройства;
* Фильтрация трафика между сегментами;
* Устаревшее у-во, заменён свичём.


## Устройства безопасности
### Firewall
* Контроль i/o трафика на основе указанных правил (Access Control List): IP, MAC.


### IDS
Intrusion Detection System.
* Мониторит трафик в поисках вирусных сигнатур;
* Ставится || адресату.


### IPS
Intrusion Prevention System.
* Мониторит трафик, блокирует передачу вирусных сигнатур;
* Ставится между адресатом и адресантом.


### UTM
Системы унифицированного управления угрозами.
Принцип «всё в одном»:
1. IPS;
2. Firewall с сохранением состояний;
3. Защита от DDoS;
4. VPN;
5. Антишпионское ПО;
6. etc.


### Сервер управления хостами
Управление всеми у-вами сети, включая ПК, телефоны etc.
* Управление устройства;
* Управление доступ к сети;
* etc.
